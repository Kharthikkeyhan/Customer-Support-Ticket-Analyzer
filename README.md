# Customer-Support-Ticket-Analyzer
## 🎯 Project Objectives

The primary objective of this project is to build a **Customer Support Ticket Analyzer** using Python that demonstrates data processing, text cleaning, and basic text analytics on customer support tickets.

The project aims to:

* Develop a ticket management system using Python dictionaries and lists to store customer support data.
* Load predefined support tickets and allow users to add new tickets dynamically through validated input.
* Perform text preprocessing by converting text to lowercase, removing punctuation, standardizing words, and cleaning extra spaces.
* Analyze customer issues by identifying keyword frequencies and extracting useful insights from ticket descriptions.
* Categorize and summarize support tickets based on priority levels (High, Medium, and Low).
* Identify the longest issue description and calculate the unique vocabulary present in the dataset.
* Generate a concise analytical report containing ticket statistics, priority distribution, sentiment keywords, and text-based insights.
* Detect text normalization issues and highlight data quality problems encountered during preprocessing.
* Demonstrate the use of fundamental Python concepts, including dictionaries, lists, loops, conditional statements, functions, string manipulation, regular expressions, and collections.
* Provide actionable recommendations that can help improve customer support operations and future enhancements of the ticket analysis system.

### Expected Outcomes

* Efficient organization and management of customer support tickets.
* Cleaner and standardized textual data for analysis.
* Better understanding of customer issue patterns through keyword and priority analysis.
* Identification of common service issues and critical support requests.
* Practical demonstration of Python-based text analytics and reporting for customer support data.
